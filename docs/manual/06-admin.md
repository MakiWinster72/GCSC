# 后台管理

> 仅 **管理员** 可访问此页面。

## 上传限制

管理员可设置个人成果、证明资料中的文件上传的数量和大小限制。

![上传限制](/images/admin上传限制.png)

## 用户管理

![用户管理页面](/images/admin用户管理.png)

### 编辑用户角色

点击用户行的「编辑」按钮，可修改用户。

![edituser](/admin/editUser.png)

角色说明：

| 角色            | 说明                                |
| --------------- | ----------------------------------- |
| STUDENT（学生） | 普通学生                            |
| CADRE（班干部） | 普通学生 + 可审核本班成员的信息修改 |
| TEACHER（教师） | 班干部 + 可查看学生档案             |
| ADMIN（管理员） | 拥有全部权限                        |

将用户改为`教师`时，可以为其分配负责的班级，这会让该账号可以获取对应班级的学生信息  
班干部账号只能获取当前账号的审核通知

![edituserTeacher](/admin/editUserTeacher.png)

## 审核策略

管理员可配置审核流程

1. 关闭审核

学生编辑信息的时候会显示“保存”，并且通知界面不会记录

1. 开启审核但自动通过

学生编辑信息的时候依然显示“申请审核”，但自动通过，通知页面有历史记录

![审核策略](/images/admin审核策略.png)

## 备份与恢复 {#backup-restore}

### 备份

可以导出 sql 文件和 zip 文件，zip 即把 uploads 文件夹打包下载。建议先检查用户附件大小检查是否有异常占用用户，清理后再备份附件

![backup](/admin/backup.png)

### 恢复

可以选择对应的 sql和zip 文件进行数据恢复
![recover](/admin/recover.png)

### 检查用户附件占用大小

![storage](/admin/storage.png)

检查各个用户使用存储情况，其实就是扫描 /uploads 文件夹下各个用户的文件夹大小，便于查看同学们对存储的使用情况，检查是否有异常占用。

每个用户行右侧有删除按钮，点击后可删除该用户在磁盘上的所有附件文件。**此操作不可恢复**，操作会被记录到审计日志中。

对于已删除用户，显示格式为 `(已删除) #文件夹名`，通过文件夹名（即用户 ID）区分不同已删除用户。

## 注册开关

管理员可开启或关闭注册功能。

![其他设置页面](/images/admin其他设置.png)

关闭后，登录界面将不再显示注册入口。
